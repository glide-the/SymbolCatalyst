## 0) 公理组（Axiom Set）

所有扩展仅允许使用本组词汇。禁止具体文化符号（地域/宗教/历史意象/具体器物/具体神话名词/具体民族/国家/时代风格/具象图腾）。

| 公理 | 形式定义 |
|------|----------|
| **精神世界** (Pre-Observation Field) | 未被捕捉与理解之前的情绪性混沌场；所有符号的前状态 |
| **工具** (Tool/Carrier) | 承载并传递意识形态的载体；被支配时呈现重复信息，因此呈现力量 |
| **意识形态** (Ideology as Iterative Structure) | 在不同工具上呈现不同结构；通过信息传递完成自我迭代 |
| **幻想** (Fantasy as Repetitive Force) | 不断重复的信息，其重复呈现力量；语言边界之外的场 |
| **对象化** (Objectification) | 通过范指动作（切片/凝固/缝合）将精神世界转换为可交流的符号与定义 |
| **符号化** (Symbolization) | 对幻想进行符号分析，生成秩序、界限、结构；警惕其赋予现实性力量 |
| **命题边界** (Proposition Boundary) | 任何规则系统必须声明适用边界与失败条件，否则即为意识形态 |
| **权力运动** (Power as Motion) | 不属于个体，表现为临时载体的运动链 |
| **主体性** (Subjectivity as Operating Force) | 在符号空间中运转的力量；以时间化的符号系统表现其运动 |

### 公理推论

- **幻想的形式色彩** = 一段时间的主体为自身存在所做的运动
- **符号的时间性** = 符号在对象化后亦是时间
- **因果-表达循环** = 因果形式决定表达方式，表达方式反向推动因果形式
- **唯一真理形式** = 保持追问（每个子词可否定，每条因果可重写）

## 1) 引擎目标

构建一个"符号系统结构与运转模式引擎"，能从输入的文本/交互数据中生成并迭代一个系统：

1. **从精神世界到符号**：把不可语言化的感受（幻想）转换成可运算的符号结构（对象化）
2. **从符号到规则**：将符号结构生成规则（命题），并附带边界与失败条件
3. **规则可改写**：规则在传播与对抗中可被改写；改写是权力运动在载体上的表现
4. **保持追问机制**：系统必须内置"追问"作为唯一真理的形式结构：每个子词可否定、每条因果可重写
5. **避免巫术**：当系统用定义反向制造原因、以因果结构填补空洞时，必须标记为"巫术风险"

## 4) 运转模式（Runtime Loop - 10阶段）

### Stage 1: 感受场进入（Field Ingestion）

**输入**: 话语流 I1
**处理**: 捕捉"未定义但重复"的片段
**输出**: `fantasy_candidates[]`

```
候选片段:
1. "师徒四人" → 重复出现的集体单位
2. "试炼/考验" → 重复出现的阻碍模式
3. "转变/正果" → 重复出现的目的论承诺
4. "向西" → 重复出现的方向性
```

**失败策略**: 话语流为空 → 输出空候选 + 沉默报告

---

### Stage 2: 切片化（Slicing）

**输入**: 幻想候选
**处理**: 提取范指单位（重复点、否定可插入点、因果连接点、边界词）
**输出**: `raw_symbols[]`

```
原始符号:
- 中心载体（慈悲+固执）
- 全能变体（强大+叛逆）
- 欲望载体（贪婪+忠心）
- 稳定载体（沉稳）
- 对立存在（阻碍力量）
- 超越存在（支持力量）
- 路径（方向性运动）
- 文本（目的论对象）
- 试炼序列（重复阻碍）
- 转变（终极承诺）
```

**失败策略**: 无可切片内容 → 降级为最小符号集（载体+路径+目的）

---

### Stage 3: 凝固（Freezing）

**输入**: 原始符号
**处理**: 检测重复使用模式，凝固为定义
**输出**: `frozen_symbols[]` + `frozen_definitions[]`

```
凝固符号:
1. "载体二元性" = 载体同时具备对立属性（慈悲/固执，强大/叛逆，贪婪/忠心）
2. "试炼递进" = 阻碍按强度和复杂度递增排列
3. "目的论担保" = 路径终点预存在起始点（取得文本是完成使命的必然结果）
4. "集体主体" = 个体载体消融于师徒共同体中
```

**凝固阈值**: 同一描述模式出现 ≥3 次

**失败策略**: 未达阈值 → 保持流动状态，标记"未凝固"

---

### Stage 4: 结构筛选（Exclusivity Filtering）

**输入**: 凝固符号
**处理**: 抽取排他机制
**输出**: `exclusivity_filters[]`

```
筛选器列表:
1. 允许：成功叙事、进步逻辑、线性时间
2. 排除：失败终局、循环停滞、路径分叉
3. 强化：集体主义、目的论、等级秩序
4. 遮蔽：个体欲望、偶然性、非线性可能
```

**失败策略**: 无排他信号 → 使用默认开放筛选器（允许所有解释）

---

### Stage 5: 符号化与秩序化（Symbolization & Ordering）

**输入**: 凝固符号 + 筛选器
**处理**: 生成符号图谱 + 幻想结
**输出**: `symbol_graph` + `fantasy_knots[]`

详见下方 **O1: Symbol Graph Snapshot**

**失败策略**: 向量化失败 → 保留文本标签，标记"未量化"

---

### Stage 6: 命题生成（Proposition Generation）

**输入**: 符号图谱
**处理**: 从图谱生成规则命题，必须附带边界与失败条件
**输出**: `propositions[]`

详见下方 **O2: Rule Deck**

**失败策略**: 无法生成边界 → 标记为意识形态风险，降级为假设

---

### Stage 7: 追问驱动（Interrogation Driver）

**输入**: 命题集合
**处理**: 对每条命题执行子词否定，评估因果形式如何反向塑造表达
**输出**: `alternative_propositions[]` + `causal_feedback_analysis`

```
示例追问:
原命题: "试炼序列必然导致终极转变"
否定1: "试炼序列不必然导致转变" → 替代解释: 试炼可能是循环或无意义
否定2: "非试炼因素导致转变" → 替代解释: 外部干预或偶然性起作用
否定3: "终极转变不是目的" → 替代解释: 过程本身即目的

因果形式分析:
- "试炼→转变"因果结构强化了线性进步叙事
- 该因果形式排除"失败-放弃-停滞"等可能表达
- 表达方式反过来固化为"成功必须经历试炼"的意识形态
```

**失败策略**: 否定后命题全部自洽 → 标记"结构封闭"警告

---

### Stage 8: 巫术检测（Witchcraft Detector）

**输入**: 命题 + 替代命题
**处理**: 检测"定义反向制造原因"的结构
**输出**: `witchcraft_scores[]` + 处置决策

详见下方 **巫术检测规则**

**失败策略**: 检测器无法判定 → 保守处理，全部标记为中风险

---

### Stage 9: 权力链改写（Power Rewrite）

**输入**: 命题 + 现实映射变量 I4 + 权力链
**处理**: 基于权力链运动改写命题的 `rewrite_hooks`
**输出**: `rewritten_propositions[]` + `rewrite_log[]`

详见下方 **O3: Rewrite Log**

**失败策略**: 无 I4 输入 → 跳过改写，保持原命题

---

### Stage 10: 输出封装（Packaging）

**输入**: 所有阶段产出
**处理**: 封装为 O1-O5 五个输出块（JSON + 可读文本双份）
**输出**: 完整引擎输出

详见下方 **O1-O5 完整输出**

**失败策略**: 部分数据缺失 → 输出已有数据 + 缺失字段报告

## 5) 巫术检测规则

### 可计算信号

| 信号 | 检测条件 | 风险增量 |
|------|----------|----------|
| 定义自指环 | 定义A引用B，B引用A，无外部验证锚点 | +0.3 |
| 因果倒置 | 结论先于前提出现在话语流中 | +0.2 |
| 边界缺失 | 命题无 boundary 或 failure_conditions | +0.4 |
| 排他性封闭 | 筛选器排除所有替代解释 | +0.3 |
| 否定不可插入 | 子词否定后命题仍完全自洽且无替代产生 | +0.2 |
| 载体单一声称 | 声称仅一种工具有效但使用跨工具词汇 | +0.1 |

### 处置规则

- `risk < 0.3` → 通过，记录风险分
- `0.3 ≤ risk < 0.6` → 强制补充边界与失败条件
- `risk ≥ 0.6` → 沉默策略（仅输出可检验问题，不输出确定性结论）

## 6) 输出风格约束

1. **只用抽象哲学词汇**，禁止具体文化符号与具体世界观元素
2. **每段必须可对应到**结构/流程/字段/算法
3. **不输出人生建议**；只输出系统结构与运转模式
4. **任何结论必须附带边界与失败条件**；否则标记为意识形态或进入沉默
