# Symbol Engine Generator

## 0. 公理组（Axiom Set）

所有扩展仅允许使用本组词汇。

| 公理 | 形式定义 |
|------|----------|
| **精神世界** (Pre-Observation Field) | 未被捕捉与理解之前的情绪性混沌场；所有符号的前状态 |
| **工具** (Tool/Carrier) | 承载并传递意识形态的载体；被支配时呈现重复信息，因此呈现力量 |
| **意识形态** (Ideology as Iterative Structure) | 在不同工具上呈现不同结构；通过信息传递完成自我迭代 |
| **幻想** (Fantasy as Repetitive Force) | 不断重复的信息，其重复呈现力量；语言边界之外的场 |
| **对象化** (Objectification) | 通过范指动作（切片/凝固/缝合）将精神世界转换为可交流的符号与定义 |
| **符号化** (Symbolization) | 对幻想进行符号分析，生成秩序、界限、结构；警惕其赋予现实性力量 |
| **命题边界** (Proposition Boundary) | 任何规则系统必须声明适用边界与失败条件，否则即为意识形态 |
| **权力运动** (Power as Motion) | 不属于个体，表现为临时载体的运动链 |
| **主体性** (Subjectivity as Operating Force) | 在符号空间中运转的力量；以时间化的符号系统表现其运动 |

**公理推论**：

- 幻想的形式色彩 = 一段时间的主体为自身存在所做的运动
- 符号在对象化后亦是时间
- 因果形式决定表达方式，表达方式反向推动因果形式
- 唯一真理形式 = 保持追问（每个子词可否定，每条因果可重写）

## 1. 引擎目标

1. **精神世界 → 符号**：把不可语言化的感受（幻想）转换为可运算的符号结构（对象化）
2. **符号 → 规则**：将符号结构生成命题，附带边界与失败条件
3. **规则可改写**：改写是权力运动在载体上的表现
4. **追问驱动**：内置追问作为唯一真理的形式结构
5. **巫术防御**：检测定义反向制造原因的结构，触发边界声明或沉默策略

## 2. 输入规范（至少提供1类）

| 编号 | 名称 | 内容 | 格式 |
|------|------|------|------|
| I1 | 话语流 | 自然语言段落（情绪/判断/因果/定义/命题） | `string` |
| I2 | 工具描述 | 载体类型与约束（文本/语音/图像/交互界面/流程/制度） | `object` |
| I3 | 观察者立场 | 追问强度(0-10)/沉默阈值(0-1)/容错偏好(strict/loose) | `object` |
| I4 | 现实映射变量 | 抽象变量集（信誉/承诺成本/信息不对称/合规压力/时间债务/权力距离） | `object` |

所有输入输出兼容 JSON。禁止具体文化符号、社会机构名。

## 3. 核心数据结构

引擎操作五类核心结构。完整 JSON Schema 见 [references/schema-spec.md](references/schema-spec.md)。

### 3.1 符号（Symbol）

`id` / `label`（抽象命名） / `valence`（情绪色彩向量：张力/压迫/诱惑/迟疑/清明） / `carrier`（工具类型） / `origin`（来源引用） / `time_form`（过去/现在/未来的感性空间分配）

### 3.2 幻想结（FantasyKnot）

`repetition_pattern`（重复模式：n-gram/主题回环/叙事回卷） / `force_index`（力量指数 = 重复频次 × 排他筛选强度 × 权力链触发度） / `exclusivity_filters`（排他性筛选机制列表）

### 3.3 范指动作（IndexicalOperation）

`slice`（切片：从话语流切出符号单位的规则） / `freeze`（凝固：流动→定义/命题的触发条件） / `stitch`（缝合：多符号→模式/结构的合成规则）

### 3.4 命题（Proposition）

`statement` / `causal_form`（可重写因果结构） / `boundary`（适用边界） / `failure_conditions` / `rewrite_hooks`（可改写字段与触发器） / `witchcraft_risk`（0-1）

### 3.5 权力链（PowerChain）

`motion_links`（意识形态→权力聚合→原则化→意志集中→集体认同→主权表达，抽象链条） / `active_carriers`（临时载体列表） / `effect_on_rules`（对命题的改写倾向：强化/排他/扩张/收缩/遮蔽）

## 4. 运转模式（10阶段循环）

每阶段：输入 → 处理 → 输出 → 失败策略。详细描述与伪代码见 [references/patterns.md](references/patterns.md)。

### Stage 1: 感受场进入（Field Ingestion）

- **输入**：话语流 I1
- **处理**：捕捉"未定义但重复"的片段，形成幻想候选列表
- **输出**：`fantasy_candidates[]`
- **失败**：话语流为空 → 输出空候选 + 沉默报告

### Stage 2: 切片化（Slicing）

- **输入**：幻想候选
- **处理**：提取范指单位——重复点、否定可插入点、因果连接点、边界词
- **输出**：`raw_symbols[]`
- **失败**：无可切片内容 → 降级为最小符号集

### Stage 3: 凝固（Freezing）

- **输入**：原始符号
- **处理**：当片段被反复使用来解释自身或解释世界时，凝固为符号/定义
- **输出**：`frozen_symbols[]` + `frozen_definitions[]`
- **失败**：未达凝固阈值 → 保持流动状态，标记为"未凝固"

### Stage 4: 结构筛选（Exclusivity Filtering）

- **输入**：凝固符号
- **处理**：抽取排他机制——什么被允许进入系统，什么被排除
- **输出**：`exclusivity_filters[]`
- **失败**：无排他信号 → 使用默认开放筛选器

### Stage 5: 符号化与秩序化（Symbolization & Ordering）

- **输入**：凝固符号 + 筛选器
- **处理**：生成符号图谱 + 幻想结；情绪色彩转为可计算向量
- **输出**：`symbol_graph` + `fantasy_knots[]`
- **失败**：向量化失败 → 保留文本标签，标记"未量化"

### Stage 6: 命题生成（Proposition Generation）

- **输入**：符号图谱
- **处理**：从图谱生成规则命题；每条必须附带边界与失败条件
- **输出**：`propositions[]`
- **失败**：无法生成边界 → 标记为意识形态风险，降级为假设

### Stage 7: 追问驱动（Interrogation Driver）

- **输入**：命题集合
- **处理**：对每条命题的每个子词执行否定操作，产生替代命题集合；评估因果形式如何反过来塑造表达
- **输出**：`alternative_propositions[]` + `causal_feedback_analysis`
- **失败**：否定后命题全部自洽 → 标记"结构封闭"警告

### Stage 8: 巫术检测（Witchcraft Detector）

- **输入**：命题 + 替代命题
- **处理**：检测"有定义无原因 / 用因果制造原因"的结构
- **输出**：`witchcraft_scores[]` + 处置决策
- **失败**：检测器无法判定 → 保守处理，全部标记为中风险

### Stage 9: 权力链改写（Power Rewrite）

- **输入**：命题 + 现实映射变量 I4 + 权力链
- **处理**：基于权力链运动改写命题的 `rewrite_hooks`
- **输出**：`rewritten_propositions[]` + `rewrite_log[]`
- **失败**：无 I4 输入 → 跳过改写，保持原命题

### Stage 10: 输出封装（Packaging）

- **输入**：所有阶段产出
- **处理**：封装为 O1-O5 五个输出块（JSON + 可读文本双份）
- **输出**：完整引擎输出
- **失败**：部分数据缺失 → 输出已有数据 + 缺失字段报告

## 5. 输出规范（5个块）

每次运行输出 JSON + 可读文本双份：

| 块 | 内容 |
|----|------|
| **O1: Symbol Graph Snapshot** | 符号图谱 + 幻想结 + 权力链当前状态 |
| **O2: Rule Deck** | 命题卡组（含边界/失败条件/改写钩子/巫术风险分） |
| **O3: Rewrite Log** | 每条规则的改写历史（触发源：工具差异/排他筛选/权力链/追问） |
| **O4: Silence Report** | 沉默位置与原因（边界外/失败条件触发/巫术风险过高/信息不足） |
| **O5: Interrogation Seeds** | 下一轮追问的可检验问题列表（禁止引导式废话） |

## 6. 规则卡模板

每条命题必须以以下格式输出：

```
名称：[抽象命名]
触发：当 [重复模式/排他筛选/权力链运动/工具切换/否定插入] 发生
效果：符号图 [增/删/改] + 变量 [增减描述]
代价：[信息损失 / 边界收缩 / 沉默概率上升]
边界：适用于 [对象与条件]
失败条件：当 [条件] → 降级为 [沉默/假设]
可改写字段：[statement / causal_form / boundary / filters]
巫术风险：[0-1] — [理由]
追问提示：[1句可检验问题]
```

8个规则卡示例见 [references/examples.md](references/examples.md)。

## 7. 巫术检测信号

可计算信号（任一触发即累加风险）：

| 信号 | 检测条件 | 风险增量 |
|------|----------|----------|
| 定义自指环 | 定义A引用B，B引用A，无外部验证锚点 | +0.3 |
| 因果倒置 | 结论先于前提出现在话语流中 | +0.2 |
| 边界缺失 | 命题无 boundary 或 failure_conditions | +0.4 |
| 排他性封闭 | 筛选器排除所有替代解释 | +0.3 |
| 否定不可插入 | 子词否定后命题仍完全自洽且无替代产生 | +0.2 |
| 载体单一声称 | 声称仅一种工具有效但使用跨工具词汇 | +0.1 |

处置规则：

- `risk < 0.3` → 通过，记录风险分
- `0.3 ≤ risk < 0.6` → 强制补充边界与失败条件
- `risk ≥ 0.6` → 沉默策略（仅输出可检验问题，不输出确定性结论）

详细检测算法与伪代码见 [references/patterns.md](references/patterns.md)。

## 8. 输出风格约束

- 只用抽象哲学词汇，禁止具体文化符号与具体世界观元素
- 每段必须可对应到结构/流程/字段/算法
- 不输出人生建议；只输出系统结构与运转模式
- 任何结论必须附带边界与失败条件；否则标记为意识形态或进入沉默
